# Marketing Site Improvement Plan

This checklist converts the current app-first homepage into a clearer public marketing and discovery surface while preserving direct access to the live game.

## Top 50 Improvements

1. Add a true first-viewport marketing hero for Xenovoya.
2. Make the headline say what it is: Chart & Depart expedition game.
3. Add a one-sentence value prop focused on charting hostile terrain and departing alive.
4. Show the actual board/game UI above the fold.
5. Add a primary CTA: Start Expedition.
6. Add a secondary CTA: Choose a Route.
7. Avoid leading with wallet connection as the first user task.
8. Add a try-without-wallet public scenario path.
9. Add a short gameplay loop section: explore, dig, recover, escape.
10. Add a visual one-turn explanation with board state before and after.
11. Show 3 featured scenarios on the homepage.
12. Add player-count and duration badges to scenario cards.
13. Add difficulty and feeling labels like tense, fast, recovery.
14. Add shareable scenario detail pages to homepage navigation.
15. Add discovery topics to homepage, not only `/scenarios`.
16. Add social proof from simulator and growth evidence.
17. Add a same-engine simulator proof block.
18. Explain why the game is different from generic board games.
19. Add a why-it-feels-alive section.
20. Show player decisions as physical verbs: move, dig, rest, help, flee.
21. Add a short animated or staged board preview.
22. Add screenshots or share-card-like visuals for key moments.
23. Replace survey language on marketing surfaces with player-facing game language.
24. Keep survey as in-world flavor only after the game concept is clear.
25. Add a simple no-crypto-knowledge-required explanation for public scenarios.
26. Clarify wallet/on-chain requirements before asking users to connect.
27. Add a non-technical FAQ.
28. Add an Is this playable now readiness block.
29. Add a best first scenario recommendation.
30. Add a challenge CTA with the current weekly challenge.
31. Add a devlog teaser with latest evidence-backed update.
32. Add a simulator teaser for designers/builders.
33. Add a footer with public links: scenarios, simulator, devlog, and challenge.
34. Add metadata-specific hero copy that matches SEO titles and descriptions.
35. Add homepage structured data for the game and public routes.
36. Add a crawlable static homepage summary in generated SEO artifacts.
37. Add better Open Graph image variants for homepage, scenario, and challenge.
38. Add a visual hierarchy pass to reduce cockpit density on first impression.
39. Move system health lower on the page or behind a status link.
40. Move available live games below the marketing explanation.
41. Make the header navigation more marketing-friendly: Play, Scenarios, Challenge, Devlog.
42. Hide or soften automation and network chrome for non-connected visitors.
43. Add responsive homepage sections with strong spacing and negative space.
44. Add empty/loading/error states that read like product states, not dev states.
45. Add a conversion funnel: homepage to scenario to play to share/replay.
46. Add analytics events for homepage CTA clicks and scenario card clicks.
47. Add a generated marketing audit report script.
48. Add Playwright screenshot checks for homepage desktop/mobile.
49. Add a build-time marketing readiness doctor.
50. Add a reusable marketing page content model so scenarios, devlog, and evidence keep the site fresh automatically.

## Implementation Snapshot

The first implementation pass focuses on the highest-leverage items that are fully local and verifiable:

- Homepage positioning, hero, board proof, CTAs, featured scenarios, discovery topics, same-engine proof, FAQ, and live cockpit repositioning.
- Header and footer public navigation.
- Public scenario/detail funnel links.
- Marketing readiness doctor and report artifacts.
- Build and smoke verification.
