# Xenovoya Game UI Design System

The living page is available at `/design-system`.

## Purpose

The Xenovoya interface should feel like a quiet expedition instrument that becomes alive when the player intends, previews, commits, waits, or resolves an action.

## Principles

- The board is the visual priority during play.
- Negative space protects gameplay readability.
- Actions are physical verbs: move, camp, dig, rest, help, flee.
- Persistent HUD stays minimal.
- Secondary information belongs in drawers, details, tabs, hover/tap details, or post-action screens.
- Motion communicates state and feedback, not decoration.
- State tones are consistent: compass for ready/primary, blueprint for planning/resolving, oxide green for recovery/success, signal red for danger/failure, relic for rare artifact beats.
